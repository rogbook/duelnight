# DuelNight 실시간 화면 검증 및 멀티 디바이스 테스트 가이드

개발자가 작성하거나 수정한 코드를 PC, 태블릿, 모바일 기기에서 실시간으로 신속하게 테스트하고 화면을 검증할 수 있는 환경 설정 가이드입니다.

---

## 1. 로컬 개발 환경에서의 실시간 테스트 (HMR)

Vite 개발 서버를 기동하면 코드를 수정할 때마다 웹 브라우저가 새로고침 없이 변경된 사항을 즉시 반영(HMR: Hot Module Replacement)합니다.

### 실행 방법

```bash
# 패키지 설치 후 개발 서버 기동
bun run dev
# 또는
npm run dev
```

터미널에 표시되는 주소 중 `Local:` 주소(예: `http://localhost:5173`)로 접속하여 PC 화면을 검사하며 개발을 진행합니다.

---

## 2. 동일 Wi-Fi 대역을 통한 모바일/태블릿 테스트

스마트폰이나 태블릿 기기가 개발용 PC와 동일한 공유기(Wi-Fi) 망에 접속되어 있다면 별도의 도구 없이 모바일 기기 브라우저로 직접 로컬 사이트에 접속할 수 있습니다.

### 단계별 절차

1. **IP 주소 확인**:
   - Windows 터미널에서 `ipconfig` 명령을 실행하여 **IPv4 주소**를 찾습니다 (예: `192.168.0.24`).
2. **모바일 접속**:
   - 모바일/태블릿 브라우저 주소창에 `http://[PC_IP]:5173`을 입력합니다 (예: `http://192.168.0.24:5173`).
3. **접속이 되지 않는 경우 (방화벽 트러블슈팅)**:
   - **Vite 설정 확인**: `vite.config.ts`에 `host: true` 옵션이 켜져 있거나 명령어를 실행할 때 `--host` 옵션을 붙여야 외부 접속이 허용됩니다 (`bun run dev --host`).
   - **Windows 방화벽 해제**: Windows Defender 방화벽이 특정 포트(5173 등)의 인바운드 연결을 차단할 수 있습니다. `제어판 > Windows Defender 방화벽 > 고급 설정 > 인바운드 규칙`에서 Node.js/Bun 또는 5173 포트에 대한 접속을 허용합니다.

## 3. 터널링 도구(ngrok)를 통한 외부 망 테스트

개발 PC와 다른 네트워크(LTE/5G 또는 외부 Wi-Fi)에 있는 태블릿이나 스마트폰에서 실시간 개발 화면을 보며 검증해야 할 때는 **ngrok**을 활용해 보안 공인 주소를 연결합니다.

```
[로컬 PC (localhost:5173)] ── ngrok 터널 ── [인터넷 공인 URL (xxxx.ngrok-free.app)] ── [모바일 기기]
```

### ngrok 설정 및 실행

1. **ngrok 전역 설치**:
   ```bash
   npm install -g ngrok
   ```
2. **터널 구동**:
   개발 서버(`bun run dev`)가 켜진 상태에서 다른 터미널을 열고 아래 명령을 실행합니다.
   ```bash
   ngrok http 5173
   ```
3. **접속 주소 확인**:
   - 터미널에 생성된 `Forwarding` 주소(예: `https://a1b2-c3d4.ngrok-free.app`)를 복사합니다.
4. **테스트 진행**:
   - 태블릿이나 모바일 기기에서 위 HTTPS 주소로 접속하면, 로컬 PC에서 구동 중인 코드가 그대로 화면에 로드되며 실시간 연동 테스트를 진행할 수 있습니다.

## 4. Cloudflare Pages Preview Deployments를 통한 팀/다기기 QA

실제 프로덕션 도메인(`duelnight.app`)에 영향을 주지 않으면서, 빌드가 완료된 정적/서버리스 환경을 독립적으로 검증하는 단계입니다.

### 4.1. 작동 흐름

1. 개발자가 작업 브랜치(`feature/UI-개선` 등)에서 작업을 마친 후 GitHub에 푸시합니다.
2. Cloudflare Pages가 저장소의 변경사항을 감지하여 자동으로 빌드를 빌드합니다.
3. 빌드가 끝나면 Cloudflare가 해당 커밋에 대한 **고유 Preview URL**을 생성합니다. (예: `https://[commit-hash].duelnight.pages.dev`)
4. 개발자, 기기 테스터는 이 Preview URL을 통해 모바일 기기, 태블릿, PC 등 다양한 해상도와 브라우저 환경에서 동시 테스트를 수행합니다.

### 4.2. 장점

- 로컬 PC를 계속 켜두거나 외부 망 방화벽 설정을 건드릴 필요가 없습니다.
- 실제 호스팅 플랫폼(Cloudflare Workers 에지 서버)에서 코드가 돌아가는 형태 그대로 검증되므로 런타임 오류를 가장 확실히 잡아낼 수 있습니다.
- 데이터베이스(Supabase)는 스테이징용 프로젝트를 가리키게 하여 프로덕션 데이터 손상 우려를 원천 차단합니다.
