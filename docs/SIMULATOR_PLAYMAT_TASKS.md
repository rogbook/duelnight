## 0. 목표 한 줄

대국 보드를 **실제 카드샵 플레이매트**처럼 — 각 카드 자리가 **윤곽선 + 라벨**로 그려져 있고, 카드를 그 자리에 놓는 느낌. "처음 보는 사람도 어디에 무엇을 놓는지 바로 안다."

## 1. 현황 (재사용 — 전면 재작성 금지)

`src/routes/simulator.$id.tsx`의 `Board` 컴포넌트(약 1170~1282행)는 이미 다음을 갖췄다. **새로 만들지 말고 이 위에 라벨·배치·시각화를 더한다:**

- 진영별 미러 배치(상대 위 / 나 아래), 진영 톤 그라디언트
- `EmptySlot`(빈 슬롯), `BattleUnit`(캐릭터), `LeaderCard`(리더), `Pile`(덱/트래시), `DonBadge`(DON 숫자), `NameBanner`(라이프 ❤️), `HandFan`/`OppHand`(손패)
- Phase 2 연출(턴/전투/KO 배너)·멀리건 오버레이·카드 확대·공격 화살표 — **전부 보존**

**부족한 것 = 이번 작업:** ① 각 존에 **라벨** 없음 ② 공식 플레이매트 **배치**가 아님 ③ DON!!이 숫자 배지뿐(코스트 에리어 토큰 아님) ④ 라이프가 숫자뿐(카드 더미 아님) ⑤ **스테이지 자리** 없음 ⑥ 자리 윤곽이 그려진 **매트 배경** 없음.

## 2. 원피스 공식 플레이매트 배치 (이 구역대로)

공식 7개 존(출처: ONE PIECE CARD GAME Play Guide):

| 존 | 위치 | 내용 |
|---|---|---|
| **리더 (Leader)** | 진영 중앙 | 리더 1장 (앞면, 시작부터) |
| **캐릭터 에리어 (Character Area)** | 리더 앞쪽(상대 방향), 한 줄 | 캐릭터 최대 5장 |
| **스테이지 (Stage)** | 리더 옆 | 스테이지 1장 (Phase1 미구현 → **빈 자리만 표시**) |
| **DON!! 덱 + 코스트 에리어** | 리더 옆(공식: 메인 덱 왼쪽) | DON!! 덱 + 액티브/레스트 DON 토큰 |
| **라이프 (Life)** | 진영 측면 | 라이프 카드 더미(겹친 카드, 리더 라이프 값만큼) |
| **덱 (Deck)** | 측면 | 메인 덱(뒷면 더미) |
| **트래시 (Trash)** | 덱 아래 | 버려진 카드 더미 |

모바일 세로 화면 배치(미러):

```
┌─ 상대(AI) ───────────────────────────┐
│            [손패 뒷면 겹침]            │
│        [캐릭터 ×5 슬롯(라벨)]          │
│  [라이프] [DON!!] [리더] [덱] [트래시] │  ← 한 줄, 라벨 달린 자리
│                          [스테이지]    │
└──────────────────────────────────────┘
┌─ 나 ─────────────────────────────────┐   ← 위아래 대칭(미러)
│                          [스테이지]    │
│  [라이프] [DON!!] [리더] [덱] [트래시] │
│        [캐릭터 ×5 슬롯(라벨)]          │
│            [부채꼴 손패]               │
└──────────────────────────────────────┘
```

## 3. 작업 항목

### 3-1. 존 라벨 + 빈 슬롯 (핵심)
- `EmptySlot`에 `label` prop 추가 → 빈 자리에 옅은 라벨 텍스트("캐릭터", "리더", "스테이지" 등)를 윤곽선 안에 중앙 표시. 채워지면 라벨 숨김.
- 점선/옅은 실선 테두리(`border-dashed border-game-line`), 자리 비율은 카드 비율(약 5:7) 유지.
- 라벨은 i18n(ko/en/ja) — 기존 `simulator` 키에 zone 라벨 추가.

### 3-2. 공식 배치 정렬
- 현재 `statusRow`(DON 배지+덱/트래시+이름배너 한 줄)를 공식 배치 줄로 재구성: **[라이프][DON!!][리더][덱][트래시]** 가로 한 줄(리더 중앙). 캐릭터 에리어는 그 줄의 상대 방향 쪽.
- `NameBanner`(나/AI + ❤️)는 매트 밖 모서리(헤더 근처)로 빼거나 라이프 더미에 붙임 — 매트 자리를 가리지 않게.

### 3-3. DON!! 코스트 에리어 시각화
- `DonBadge`(숫자) → **코스트 에리어**: 액티브 DON은 세워진 토큰, 레스트 DON은 눕힌(회전) 토큰 작은 더미로. 총량/액티브 수가 한눈에. DON!! 덱 잔량(`donDeck`)도 작게.
- 토큰은 lucide `Coins` 또는 단순 원형 칩 반복. 과한 렌더 금지(수십 개여도 가볍게 — 최대 표시 후 "+N").

### 3-4. 라이프 카드 더미
- `NameBanner`의 ❤️ 숫자에 더해, **라이프 존에 겹친 카드 더미**(뒷면, life 수만큼, 최대 N장 겹침 표시). 피격 시 한 장 줄어드는 게 보이도록(Phase 2 damage 연출과 연결).

### 3-5. 스테이지 빈 자리 + 매트 배경
- 스테이지 자리: 라벨 단 빈 슬롯만(Phase 1 미구현이라 카드 안 올라감).
- 매트 배경: 진영별 그라디언트(현행) 위에 **자리 윤곽이 그려진 플레이매트** 느낌(옅은 라운드 패널/구분선). 다크 게이머 톤 유지.

## 4. 불변 규칙 (위반 시 반려)

- **엔진·DSL·게임 로직·액션 흐름 변경 금지.** 표현(레이아웃·라벨·시각화)만. `gameState` 읽기만, 쓰기 금지.
- Phase 2 자산(연출·멀리건·카드 확대·공격 화살표·손패) **전부 보존**. 멀리건 오버레이/결과 화면과 겹치지 않게.
- 새 패키지 금지, lucide만. game 토큰(`--game-*`) 사용, 라이트 테마는 깨지지만 않게.
- i18n 신규 문구는 ko/en/ja 3개 모두(`useI18n` 패턴, 컴포넌트 로컬 하드코딩 금지).
- 가로 좁은 모바일에서 5캐릭터 슬롯이 넘칠 때 가로 스크롤 허용(현행 유지), 자리가 찌그러지지 않게.
