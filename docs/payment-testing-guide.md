# 결제 테스트 가이드 (Test Mode)

이 문서는 DuelNight의 결제 시스템을 실제 결제 없이 테스트하는 방법을 설명합니다.

---

## 1. 테스트 모드 개요
결제 팝업 상단에 있는 **Test Mode** 스위치를 통해 테스트와 실결제 모드를 전환할 수 있습니다.
- **ON (활성화)**: 실제 돈이 빠져나가지 않는 가상 결제 모드입니다.
- **OFF (비활성화)**: 실제 결제가 진행되는 운영 모드입니다.

---

## 2. 국내 결제 테스트 (PortOne)

현재 테스트 모드(ON)에서는 **카카오페이 테스트 PG**가 기본으로 설정되어 있습니다.

### 테스트 방법:
1. 상점에서 상품 선택 후 결제창 열기
2. **Test Mode**가 ON인지 확인
3. **국내 결제** 버튼 클릭
4. 카카오페이 결제창이 뜨면 휴대폰 번호와 생년월일을 입력하여 가상 결제 진행 (실제 청구되지 않음)

### 테스트용 식별코드:
- 현재 테스트 모드에서는 포트원에서 제공하는 공용 테스트 ID(`imp31011697`)를 사용하도록 설정되어 있습니다.
- 본인의 포트원 계정에서 테스트하려면 관리자 콘솔의 '가맹점 식별코드'를 `PaymentDialog.tsx`의 `userCode` 부분에 업데이트하세요.

---

## 3. 해외 결제 테스트 (PayPal)

PayPal 테스트는 PayPal Sandbox 계정이 필요합니다.

### 테스트 준비:
1. [PayPal Developer](https://developer.paypal.com/) 접속
2. **Testing Tools > Sandbox Accounts**에서 구매자(Personal)와 판매자(Business) 계정 생성
3. **Apps & Credentials**에서 Sandbox용 Client ID 발급

### 테스트 방법:
1. 결제창에서 **해외 결제** 선택
2. 생성한 Sandbox 구매자 계정으로 로그인하여 결제 진행

---

## 4. 테스트 체크리스트

테스트 시 다음 사항들을 확인하세요:
- [ ] 결제 완료 후 `onSuccess` 콜백이 정상적으로 호출되는가?
- [ ] 결제 취소 시 에러 메시지가 사용자에게 적절히 표시되는가?
- [ ] (추후 구현 시) 결제 완료 후 DB의 크레딧 정보가 자동으로 업데이트되는가?
- [ ] 모바일 환경에서도 결제창이 정상적으로 열리고 닫히는가?

---

## 5. 운영 전환 시 주의사항
실제 서비스를 오픈할 때는 다음을 반드시 확인해야 합니다:
1. `PaymentDialog.tsx`의 `isTest` 초기값을 `false`로 변경하거나, 환경 변수(`VITE_IS_PRODUCTION`)에 따라 분기되도록 수정합니다.
2. 포트원 가맹점 식별코드를 테스트용(`imp31011697`)이 아닌 **실제 발급받은 코드**로 교체합니다.
3. PayPal Client ID를 **Live용 ID**로 교체합니다.
