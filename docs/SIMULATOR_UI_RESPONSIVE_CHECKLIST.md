# 시뮬레이터 UI 반응형 체크리스트 (Lovable 작업용)

> 대상: `src/routes/simulator.index.tsx`(로비/덱), `src/routes/simulator.$id.tsx`(배틀), 효과 편집기.
> 목표: **데스크톱 · 태블릿 · 모바일(앱 포함)** 모두에서 끊김 없이 동작.
> 검증은 Lovable 미리보기에서 3폭(모바일 375px / 태블릿 768px / 데스크톱 1280px) 직접 확인.

## 0. 공통 원칙 (Tailwind)

- [ ] 기준 브레이크포인트: 모바일 기본 → `sm`(≥640) → `md`(≥768, 태블릿) → `lg`(≥1024, 데스크톱).
- [ ] 컨테이너: `mx-auto w-full max-w-6xl px-4 sm:px-6`. 가로 스크롤 금지(`overflow-x-hidden` 필요 구간만).
- [ ] **터치 타깃 최소 40~44px**(`h-10`+). 모바일에서 너무 작은 아이콘 버튼 지양.
- [ ] 폰트: 모바일 `text-xs/sm` → 데스크톱 `sm:text-base`. 고정 px 폭 지양, `min-w-0`로 truncate 허용.
- [ ] 안전영역(모바일앱): 하단 고정 바는 `pb-[env(safe-area-inset-bottom)]`.
- [ ] **다크/라이트/시스템** 3모드 모두에서 대비 확인(토큰 `bg-card/foreground/muted` 사용, 하드코딩 색 지양).

## 1. 로비 / 덱 목록 (`simulator.index.tsx`)

- [ ] 덱 카드 그리드: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` (모바일 1열).
- [ ] 상단 액션(새 덱/모드 선택 등): 모바일 `flex-col`/wrap, 데스크톱 가로 정렬.
- [ ] 덱 빌더(있다면): 카드 리스트 + 선택 패널을 모바일은 **세로 스택**, `lg`부터 2-패널 좌우 분할.
- [ ] "보유 카드만 보기" 토글 등 필터: 모바일에서 한 줄에 안 들어가면 wrap.
- [ ] 긴 덱 이름/리더명 `truncate`.

## 2. 배틀 화면 (`simulator.$id.tsx`) — 가장 중요

보드(상대 진영 ↔ 내 진영)는 가로가 넓어 모바일에서 깨지기 쉬움.

- [ ] **세로 레이아웃 우선**: 상대 보드(위) / 중앙 정보 / 내 보드(아래)를 세로 스택. 데스크톱도 동일 축(위=상대, 아래=나)이 자연스러움.
- [ ] **벤치/캐릭터 영역**: 카드들을 `flex gap-1 overflow-x-auto`(가로 스와이프) 또는 `grid grid-cols-3 sm:grid-cols-5`. 모바일에서 5장이 한 줄에 안 들어가면 가로 스크롤 허용.
- [ ] **카드 크기 반응형**: `w-12 sm:w-16 lg:w-20` 식으로. 종횡비 `aspect-[5/7]` 고정.
- [ ] **손패(hand)**: 모바일은 화면 하단 가로 스크롤 트레이, 데스크톱은 부채꼴/가로 정렬. 카드 수 많아도 `overflow-x-auto`.
- [ ] **데미지/라이프/돈 카운터**: 숫자는 항상 보이게(축소돼도 가독). 작은 화면에서 아이콘+숫자 묶음 wrap.
- [ ] **액션 버튼/턴 진행 바**: 모바일은 **하단 고정**(`sticky bottom-0` + safe-area), 데스크톱은 우측/하단. 버튼 터치 타깃 확보.
- [ ] **AI 속도/자동진행 컨트롤**: 모바일에서 한 줄 wrap, 슬라이더/셀렉트 폭 `w-full sm:w-auto`.
- [ ] **로그/히스토리 패널**: 데스크톱 우측 사이드, 모바일은 접이식(드로어/`details`) 또는 하단 시트.
- [ ] 카드 클릭 시 상세/확대: 모바일은 풀스크린 다이얼로그, 가로 회전 대응.
- [ ] 텍스트 오버플로 없음(효과 텍스트 `line-clamp` + 탭/모달로 전체 보기).

## 3. 효과 편집기 (`card-effects-editor.tsx`) — 이미 반응형, 점검만

- [ ] 스니펫 버튼 `flex-wrap`(확인됨), textarea `w-full`(확인됨).
- [ ] 모바일에서 textarea 높이 적당(rows=8), 코드 폰트 가독.

## 4. 회귀 점검 (모드별)

- [ ] 모바일 375px: 가로 스크롤 없음(보드 내부 스와이프는 의도된 것만), 버튼 안 잘림.
- [ ] 태블릿 768px: 2열/2패널 전환 자연스러움.
- [ ] 데스크톱 1280px: 여백 과다 없이 보드가 중앙 정렬.
- [ ] 모바일 앱: 하단 안전영역 겹침 없음, 상단 노치 영역 침범 없음.
- [ ] 다크/라이트/시스템 3모드 대비 OK.

## 우선순위

1. **배틀 화면 세로 스택 + 보드 가로 스크롤**(모바일 핵심)
2. 손패 트레이 + 하단 고정 액션바(safe-area)
3. 로비/덱 그리드 1→2→3열
4. 로그 패널 접이식

> 비고: 엔진/AI 로직은 건드리지 말고 **레이아웃/클래스 위주**로. 검증은 미리보기 3폭 + 3테마.
